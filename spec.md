# Open Assets

Open assets is inspired by the screenshots folder in cat-iq-test/screenshots.

It makes it easy to generate and export assets for launching apps and websites in various locations

You install open assets in your project and run it like Storybook where you can see live updating previews with hot reload. You may prompt to make changes. All the html is hosted in your repository. 

It is meant to be installed in monorepos

## Product

### Features
- You may configure export locations i.e. when you make changes to your app icon
- Components -- components may be either html or SVGs or any other type of image you may reference reusable components. you may reference them in
- Assets
    - you may provide assets that may be used
- Templates -- you may create templates which make it easy to generate new assets -- i.e. app store screenshot
- you may write directory to screenshot directories via UI stests
- automations -- the openassets CLI allows you to render / export assets and publish them to your App Store listing with no additional steps
- Canvas -- the canvas runs on port <PORT> --
